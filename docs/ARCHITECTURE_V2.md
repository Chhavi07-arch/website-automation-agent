# Architecture V2 — Goal-Oriented Planning Layer

## What changed in V2

Version 1 workflows called browser tools directly:

```
Workflow ──────────────────────────────────── Tools
  agent.scroll.scrollDown(600)
  agent.formDetection.detectFields()
  agent.input.fill(locator, value)
```

Version 2 inserts a **Planner** between the Workflow's intent and the
ActionExecutor's dispatch:

```
Workflow
   │  declares goal + params
   ▼
Planner.generatePlan(goal, params)
   │  returns ordered action[]
   ▼
ActionExecutor.executeAll(plan)
   │  resolves field names → Locators, calls tools
   ▼
Tools (NavigationTool, InputTool, ClickTool, …)
   │
   ▼
Playwright → Browser
```

---

## The Goal → Plan → Execute → Verify flow

### 1. Goal (Workflow layer)

A workflow declares *what* it wants done in 3 lines:

```js
const plan = agent.generatePlan('FILL_SHADCN_FORM', {
  url, nameValue, descriptionValue,
});
await agent.executor.executeAll(plan);
```

The workflow has no knowledge of scroll amounts, timeouts, or CSS selectors.

### 2. Plan (Planner layer)

`Planner.generatePlan()` translates the goal into an ordered array of typed
action objects.  The array is **pure data** — no Playwright objects, no
side effects.  It can be:

- Logged in its entirety before any browser action runs (full transparency).
- Serialised to JSON and replayed.
- Generated by an LLM in Phase 4 (just replace `generatePlan()` body).

Example plan for `FILL_SHADCN_FORM`:

```
Step 01: Navigate → https://ui.shadcn.com/...
Step 02: Screenshot [after-navigation]
Step 03: Wait for network idle
Step 04: Wait 1500ms
Step 05: Scroll down 600px
Step 06: Screenshot [before-form-fill]
Step 07: Detect field "name"
Step 08: Detect field "description"
Step 09: Click field "name"
Step 10: Fill "name" → "Jane Doe"
Step 11: Verify "name" === "Jane Doe"
Step 12: Click field "description"
Step 13: Fill "description" → "..."
Step 14: Verify "description" === "..."
Step 15: Screenshot [after-form-fill]
```

### 3. Execute (ActionExecutor layer)

`ActionExecutor.executeAll(plan)` iterates the plan and dispatches each step
to the appropriate tool.  Two V2 additions make this work with symbolic field
names:

**Field Registry** — A `Map<string, Locator>` inside the executor.  When a
`DETECT_FIELD` step runs, `FormDetectionService.detectFields()` is called once
(lazily, then cached).  All subsequent `FILL`/`CLICK`/`VERIFY_FIELD` steps
resolve field names from this registry instead of carrying raw Playwright
Locators in the plan.

**Step logging** — Each step is logged as `[i/total] ACTION_TYPE → "field"`
so the console shows execution progress against the printed plan.

### 4. Verify (ValidationService)

`VERIFY_FIELD` steps call `ValidationService.fieldHasValue()` after every fill,
confirming the DOM reflects the intended value.  Mismatches are logged as
warnings; the plan continues.

## Why the Planner exists

| Without Planner | With Planner |
|-----------------|--------------|
| Workflow mixes "what" and "how" | Workflow only declares goal |
| Agent intent visible only in hindsight | Full plan logged before first action |
| Adding a new goal = editing the workflow | Adding a goal = adding a private method in Planner |
| Hard to unit-test without a browser | Plan array is pure data, testable without Playwright |
| LLM integration requires workflow rewrite | Swap `generatePlan()` body, nothing else changes |

## Adding a new goal (3 steps)

1. Add the key to `ACTION_TYPES.GOALS` in `src/config/constants.js`:
   ```js
   GOALS: { FILL_SHADCN_FORM: '...', SEARCH_GITHUB: 'SEARCH_GITHUB' }
   ```

2. Add a private `_planSearchGithub(params)` method in `Planner.js` that
   returns an action array.

3. Register it in the `GOAL_MAP` inside `generatePlan()`:
   ```js
   SEARCH_GITHUB: () => this._planSearchGithub(params),
   ```

No other file needs to change.

---

## Phase 4: AI Integration point

When an LLM reasoning layer is added, `Planner.generatePlan()` becomes:

```js
async generatePlan(goalKey, params) {
  const prompt = this._buildPrompt(goalKey, params);
  const response = await llm.complete(prompt);          // Claude API call
  const plan = JSON.parse(response);                    // parse action[]
  this._logPlan(plan);
  return plan;
}
```

The ActionExecutor, Tools, Services, and Workflows remain **unchanged**.
The Planner is the single seam where AI enters the system.
