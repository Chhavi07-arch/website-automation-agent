## Recruiter-facing project description

**Website Automation Agent** — *Node.js · Playwright · Winston · OpenRouter*

A browser-automation **agent framework** (not a script) demonstrating clean layered
architecture, agent design patterns, and production-minded engineering. It detects
page elements via accessibility heuristics, executes pure-data action plans with
retries and self-healing recovery, classifies every run as SUCCESS / BLOCKED /
FAILED with diagnostics and HTML reports, and can turn an English goal into a
runnable plan via an LLM acting strictly as a planner (gated by a quality reviewer,
with offline fallback). Quality is measured by a 20-goal benchmark and 49 automated
tests. Adding a new task is a JSON file; adding an AI brain required no executor change.

## LinkedIn project description

**Website Automation Agent** — an intelligent, modular browser-automation agent
framework (Node.js + Playwright), inspired by Browser Use.

Highlights:
- 🧠 Observe → Think → Act → Verify loop with colour-coded reasoning logs.
- 🎯 Goal-routed workflows + a JSON-driven multi-step engine (variables + conditionals) — add tasks without code.
- 🔁 Resilience: exponential-backoff retries, a self-healing detection recovery ladder, bounded navigation retries.
- 🎯 Honest outcomes: SUCCESS / BLOCKED (anti-bot) / FAILED with exit codes, JSON diagnostics, and HTML run reports.
- 🤖 AI planner (OpenRouter) — natural language → validated, reviewer-scored task JSON; planner-only, with offline fallback.
- 📊 A 20-goal benchmark suite + 49 automated tests (9 suites).

Designed in clean layers (Tools → Services → Agent → Workflows) with configuration
management, structured logging, robust error handling, and a stable execution engine
that an AI planner plugs into without modification.
